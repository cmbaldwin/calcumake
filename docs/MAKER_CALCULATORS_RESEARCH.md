# Maker-Oriented Calculators Research

**Date:** 2026-01-16
**Purpose:** Research maker-oriented calculators from omnicalculator.com to implement as free tools on CalcuMake

## Overview

This document catalogs maker-oriented calculators found on [OmniCalculator.com](https://www.omnicalculator.com), organized by category. Each calculator represents an opportunity to provide valuable free tools that drive traffic to CalcuMake while serving the maker community.

## 1. Woodworking Calculators

### Board Foot Calculator
- **URL:** https://www.omnicalculator.com/construction/board-foot
- **Purpose:** Calculate the volume of hardwood lumber in board feet
- **Formula:** `board_feet = length(ft) × width(in) × thickness(in) / 12`
- **Use Cases:** Lumber purchasing, project cost estimation
- **Priority:** HIGH - Core woodworking calculation

### Lumber Calculator
- **URL:** https://www.omnicalculator.com/construction/lumber
- **Purpose:** Calculate volume, total length, and cost of lumber
- **Use Cases:** Material ordering, budget planning
- **Priority:** HIGH - Complements board foot calculator

### Lumber Weight Calculator
- **URL:** https://www.omnicalculator.com/construction/lumber-weight
- **Purpose:** Calculate weight and volume for various wood types
- **Use Cases:** Shipping calculations, structural planning
- **Priority:** MEDIUM - Specialized need

## Technical Architecture Notes

### Calculator Layout Pattern
All calculators should follow the existing advanced calculator pattern:
- Clean, mobile-responsive interface
- Real-time calculations
- Input validation
- Unit conversions where applicable
- Export functionality (PDF/CSV where relevant)
- Auto-save to localStorage
- Strategic CTAs for account creation

### API Structure
Each calculator should expose:
- RESTful API endpoint (`/api/v1/calculators/{calculator-name}`)
- JSON request/response format
- Rate limiting (public endpoints)
- Optional authentication for enhanced features
- Comprehensive OpenAPI documentation

### Documentation Strategy
Dual documentation approach:
1. **Web documentation** - Interactive guides on CalcuMake.com
2. **Markdown files** - Technical API documentation for developers
